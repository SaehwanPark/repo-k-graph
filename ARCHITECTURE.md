# ARCHITECTURE

## Overview

`repo-k-graph` (`rkg`) is a Rust project for deterministic repository knowledge graphs and context infrastructure for AI-assisted software engineering.

- `crates/rkg-lang-rust`: Rust parser integration crate.
  - current state: Tree-sitter-backed Rust parsing with deterministic symbol extraction (modules, functions, structs, enums, traits, impl blocks, and methods), import extraction (from `use` declarations), implementations (from `impl` blocks), method/function calls (from call expressions), type references (from parameters, return types, field types), test discovery (cfg-test, #[test], #[tokio::test], #[test_case], #[rstest]), macro expansion simulations (Serde, Thiserror, Clap derive), Tokio concurrency topology extraction (spawns, channels, and selects), web framework route/extractor parsing (Axum/Actix-web route chains, attribute macros, State/Data/Json extractors, and model-like Path extractors), and Rust safety extraction (unsafe blocks/functions/traits and FFI bindings).
- `crates/rkg-lang-fsharp`: F# parser integration crate.
  - current state: Tree-sitter-backed F# parsing with deterministic symbol extraction (namespaces, modules, record types, discriminated unions, classes, interfaces, active patterns, let-bound values/functions, and class members), relationship extraction (module opens, function/value references capturing pipelines, composition, computation expressions, and type references), test discovery (xUnit/NUnit attribute tests, and Expecto DSL test blocks), and web framework route extraction (Giraffe pipeline routes and Saturn router builder routes).
- `crates/rkg-lang-mojo`: Mojo parser integration crate.
  - current state: Tree-sitter-backed Mojo parsing using the repo-local grammar binding with deterministic symbol extraction (modules, classes, structs, traits, functions, and methods), import extraction, call extraction, type reference extraction, Python interop import references, and test discovery.
- `crates/rkg-lang-kotlin`: Kotlin parser integration crate.
  - current state: Tree-sitter-backed Kotlin parsing using the modern `tree-sitter-kotlin-ng` grammar with deterministic symbol extraction (packages, classes, objects, companion objects, interfaces, functions, methods, and properties), relationship extraction (imports, receiver-aware calls, type references, class/interface inheritance, annotations, and helper calls inside supported Flow pipelines), Ktor route extraction (routing path groupings and terminal HTTP methods), generated-code provenance tracking (KSP, KAPT, or generic generated source paths), annotation resolution of generated symbols back to annotated source classes (serializers, DAOs, Dagger components, Moshi adapters), and multi-source Flow topology (`combine`, `zip`, `collectLatest`, `collectIndexed`, `launchIn`).
- `crates/rkg-lang-swift`: Swift parser integration crate.
  - current state: Tree-sitter-backed Swift parsing using the modern `tree-sitter-swift` grammar with deterministic symbol extraction (modules, structs, classes, enums, protocols, extensions, functions, properties, and type aliases), relationship extraction (imports, calls, type references, protocol conformances, extensions, and attributes), test discovery (XCTestCase classes/methods and `@Test` annotations), concurrency topology extraction (`Task`, `Task.detached`, `AsyncStream`, `for await` loops), and SwiftUI/UIKit UI-bindings and macro depth (including property wrappers like `@State` and `@IBOutlet`, methods like `@IBAction`, nested view compositions inside View bodies, `#Preview` freestanding macro invocations, and storyboard/nib string reference mappings to virtual symbols).
- `crates/rkg-mcp`: Model Context Protocol (MCP) Server integration, exposing tools (`find_symbol`, `get_symbol`, `get_callers`, `get_callees`, `get_docs`, `get_tests`, `get_impact_analysis`, and `get_context_pack`) to AI coding agents over standard MCP stdio transport.

## Data Flow

Last Reviewed: 2026-06-07
Status: Verified

Current data flow includes local database lifecycle commands, repository ingestion, symbol extraction, relationship querying, framework metadata, concurrency topology, safety profile, and coverage profile querying. The broader runtime flow is:

1. Repository files are discovered by an ingestion layer.
2. Language adapters extract structured symbols, relationships, tests, docs, routes, concurrency, safety, and coverage facts.
3. Facts, relationships, and generated-symbol records are stored in the local SQLite knowledge store.
4. Previously unresolved import edges are automatically batch-resolved, and generated symbols are resolved to their source classes.
5. Unified query logic returns deterministic repository relationships.
6. CLI and MCP surfaces expose the same underlying query behavior.

## Boundaries And Constraints

Last Reviewed: 2026-06-07
Status: Verified

- Keep deterministic repository facts separate from agent reasoning.
- Keep shared domain types in `rkg-core`; avoid persistence, CLI, or MCP dependencies there.
- Keep CLI and MCP as interfaces over shared query logic rather than duplicating query behavior.
- Prefer small, typed crates over a monolithic binary as implementation grows.
- Preserve spec-driven updates in `SPEC.md`, `ARCHITECTURE.md`, and `CHANGELOG.md` when project behavior or structure changes.
